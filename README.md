# Engineering Playbook

A personal engineering playbook that documents how I learn, build software, collaborate with AI, and organize software projects.

This repository is designed to be the single source of truth for my development workflow. Instead of rewriting the same instructions for every project, I maintain a set of reusable standards, templates, and collaboration guides that evolve over time.

The goal is not only to improve code quality, but also to build consistent engineering habits, better documentation, and an efficient AI-assisted workflow.

---

# Objectives

This repository helps me:

- Learn programming more effectively.
- Keep consistent coding standards across projects.
- Document projects in a structured way.
- Collaborate efficiently with AI assistants.
- Reduce repetitive setup work.
- Continuously improve my engineering workflow.

---

# Repository Structure

```text
AI/
    AI collaboration guides.

Learning/
    Learning methodology, mentoring style, and study workflow.

Development/
    Engineering standards, project organization, debugging, Git workflow, and coding practices.

Templates/
    Reusable templates for project documentation.

Evolution/
    Long-term improvements and repository evolution.
```

---

# Philosophy

This repository follows a simple principle:

> Build once. Improve continuously.

Every document starts as a practical version, is tested in real projects, and is refined through experience instead of being over-engineered from the beginning.

The focus is consistency, clarity, and continuous improvement.

---

# How This Repository Is Used

Before starting a new project:

1. Read the relevant AI guides.
2. Follow the documented development workflow.
3. Use the provided templates when documentation is required.
4. Update the playbook whenever a better workflow is discovered.

This repository evolves together with my experience as a software developer.
